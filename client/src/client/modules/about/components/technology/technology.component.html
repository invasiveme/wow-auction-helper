The web client

The client is created using the latest version of Angular.

The application uses Angular Material combined with Bootstrap 4 for it's styling. Some custom modifications has been made also. For icons I've used Font Awesome.

The charts in the app is created with Charts.js.

The client is hosted in an AWS S3 bucket.

Server side

The backend comes in two parts. One that handles anything user related, and one that handles auction data, items etc.

User data backend

AWS Cognito: Handles user login and sessions. The front-end uses AWS amplify to communicate with it.

AWS AppSync: Handles user settings, and syncronizes them between devices and browsers.

Game data backend

The current backend is written with NodeJS with TypeScript, and is created as AWS Lambdas. This is the third iteration of the back-end. Originally it was in PHP, then NodeJS with ExpressJS, and now it's moved to AWS Lambda.

Storage

DynamoDB(NoSQL): Is used for keeping track of when auction houses last were updated, and similar data. I am using it for this, as it is serverless, fast, cheap, and will not cause issues with too many concurrent connections.

MariaDB(RDS): The item price history is stored in a RDS database, as that is what makes sense for this kind of data. It is only the backend that adds data to this database, and the insertion of new price data is queued in order to not loose price history. The price history is stored per month for 6 day, and per hour for 14 days. The RDS database also stores, recipes, items etc. But the user will never get the data directly from here, but instead via AWS S3 as that is faster.

AWS Simple Storage Solution (S3): Is used to store any static data. Items, professions, the client, auction data, tsm data etc. AWS S3, is really fast and also scales really well. Raw auction data is stored for 14 days. The app also uses to store queued item price history queries, and once a query is processed it's deleted from S3.
